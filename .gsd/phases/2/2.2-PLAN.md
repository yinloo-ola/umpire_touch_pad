# Plan 2.2: Frontend Match Detail Editing

## Objective
Update the `MatchDetailView.vue` in the Admin Dashboard to support an edit mode, enabling admins to modify the match status, scores, games, and add manual remarks.

## Context
- .gsd/SPEC.md
- .gsd/DECISIONS.md
- .gsd/phases/2/RESEARCH.md
- frontend/src/views/admin/MatchDetailView.vue

## Tasks

<task type="auto">
  <name>Edit Mode State & Override Confirmation</name>
  <files>
    frontend/src/views/admin/MatchDetailView.vue
  </files>
  <action>
    - Add an "Edit Match" button to the header area.
    - Set up reactive state: `isEditing` (boolean), `editForm` (deep copy of match status, games list, and a new `remarks` string).
    - When clicking "Edit Match": if `matchData.match.status === 'in_progress'`, show a native `window.confirm("This match is currently live. Force Override?")`. If canceled, abort. Otherwise, enter edit mode.
  </action>
  <verify>grep -q "isEditing" frontend/src/views/admin/MatchDetailView.vue</verify>
  <done>Edit Mode toggle works and prompts for in-progress matches.</done>
</task>

<task type="auto">
  <name>Editable Form UI</name>
  <files>
    frontend/src/views/admin/MatchDetailView.vue
  </files>
  <action>
    - In Edit Mode, swap out the static Status badge for a `<select>` with match statuses (unstarted, warming_up, in_progress, completed).
    - Swap static game scores for `<input type="number">` fields bound to `editForm.games[idx]`.
    - Add a "Delete" (trash icon or text) button next to each game in edit mode to remove it from the `editForm.games` array.
    - Add an "Add Game" button below the table that pushes a new game object with `0-0` scores.
    - Add a "Remarks (Force End/Retirement Reason)" `<textarea>` visible in edit mode.
  </action>
  <verify>grep -q "<input type=\"number\"" frontend/src/views/admin/MatchDetailView.vue</verify>
  <done>Form UI displays appropriate interactive elements during edit mode.</done>
</task>

<task type="auto">
  <name>Submission & Error Handling</name>
  <files>
    frontend/src/views/admin/MatchDetailView.vue
  </files>
  <action>
    - Add "Save Changes" and "Cancel" buttons in Edit Mode.
    - "Save Changes" should `PUT` to `http://localhost:8080/api/admin/matches/${matchId}` with the `AdminMatchUpdateRequest` payload `{ status, remarks, games }`.
    - If the backend returns an error (e.g. 400 Bad Request regarding score validation), catch and display it as an inline alert or error message within the component.
    - On success, exit edit mode, clear errors, and re-fetch `load()` to reload the canonical state from the server.
  </action>
  <verify>grep -q "/api/admin/matches" frontend/src/views/admin/MatchDetailView.vue</verify>
  <done>Saving applies changes cleanly and handles validation errors</done>
</task>

## Success Criteria
- [ ] Admin can enter edit mode, modifying game scores and adding/removing games.
- [ ] Attempting to edit an `in_progress` match prompts for confirmation.
- [ ] Submitting the changes hits the new `.api/admin/matches` endpoint.
- [ ] Any validation failures (like 11-10 with no remark) are displayed to the user gracefully.
