# Phase 1: Clean Architecture + `sqlc` Research

## Objective
To implement Phase 1 (Database + Core APIs `GET /matches` & `POST /match`) using a Clean Architecture pattern, driven by `sqlc` to generate the sqlite bindings.

## `sqlc` Setup Configuration
To use `sqlc`, we need:
1. A configuration file `sqlc.yaml`.
2. A schema definition file (e.g., `backend/db/schema.sql`).
3. Query definitions (e.g., `backend/db/query.sql`).
4. Generating the go code: `cd backend && sqlc generate`.

### Sample `sqlc.yaml`
```yaml
version: "2"
sql:
  - engine: "sqlite"
    queries: "db/query.sql"
    schema: "db/schema.sql"
    gen:
      go:
        package: "store"
        out: "internal/store"
        emit_json_tags: true
        emit_prepared_queries: false
        emit_interface: true
        emit_exact_table_names: false
```

### Clean Architecture Directory Layout
```
backend/
├── cmd/
│   └── server/
│       └── main.go       # Wire dependencies, mux, connect DB, configure CORS
├── db/
│   ├── schema.sql        # Table definitions: matches, games, cards
│   └── query.sql         # SQL queries: createMatch, getMatches, etc.
├── internal/
│   ├── api/
│   │   └── handlers.go   # HTTP endpoints decoding JSON, calling services
│   ├── service/
│   │   └── match_svc.go  # Business logic mapping API structures <-> DB models, timezone filtering
│   ├── store/
│   │   ├── ...           # Generated by sqlc (models.go, query.sql.go, querier.go)
│   │   └── db.go         # Generated by sqlc + any local connection wrapper if needed
```

## Date Filtering (Timezone Aware)
The requirement specifies that `GET /matches` should return matches for "today", timezone aware.
SQLite doesn't natively handle advanced time zones well, but `modernc.org/sqlite` handles standard ISO8601 strings.
In Go, we can pass `time.Now()` at the correct location bounds (Start of Day, End of Day) down to the query.
Query might look like:
```sql
-- name: GetUnstartedMatchesForPeriod :many
SELECT * FROM matches 
WHERE status = 'unstarted' 
  AND scheduled_date >= ? 
  AND scheduled_date <= ?;
```
The Service layer will compute the local `startOfDay` and `endOfDay` UTC strings (or Unix time) and pass them to the Store layer.

## Type Mappings
The `matches` table maps players into flat fields (`team1_p1_name`, etc.). 
The Service layer takes `[]Player` from the API and correctly populates the flat DB fields on insertion.
On retrieval, the Service layer scans the flat fields and packs them back into the `[]Player` structure inside a `Match` response.

## Conclusion
We will split Phase 1 into 3 atomic plans:
- **1.1:** Setup `sqlc` config, define schema/queries, and generate datastore interfaces.
- **1.2:** Build the Core Backend Wiring: `cmd/server/main.go`, ENV DB connection string setup (`modernc.org/sqlite`), and the `internal/store` linkage.
- **1.3:** Build the HTTP API & Service Layers: `api/` and `service/` logic to connect `POST /match` and `GET /matches`.
